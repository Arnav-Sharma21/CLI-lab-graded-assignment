Question 4 Commands Log
=======================

1. Command: cat > io_test.sh << 'EOF'
   #!/usr/bin/env bash
   echo "STDOUT message"
   echo "STDERR message" >&2
   sleep 30
   EOF
   Output:
   [No output or file created successfully]
   Explanation: Creates the test script 'io_test.sh' which writes to both standard output and standard error, then sleeps for 30 seconds.

2. Command: chmod +x io_test.sh
   Output:
   [No output or permissions modified successfully]
   Explanation: Marks 'io_test.sh' as executable.

3. Command: ./io_test.sh > out.log 2> err.log &
   Output:
   [1] 3628
   Explanation: Executes the script in the background, redirecting standard output (stdout) to out.log and standard error (stderr) to err.log. The shell returns job number [1] and PID 3628.

4. Command: jobs
   Output:
   [1] + running      ./io_test.sh > out.log 2> err.log
   Explanation: Displays background jobs, confirming that job [1] (PID 3628) is currently running.

5. Command: ps aux | grep io_test.sh
   Output:
   [1] + 3628 done       ./io_test.sh > out.log 2> err.log
   Explanation: Checks active process list using ps. The output shows that the background job has completed (indicated by "done").

6. Command: ps aux | grep io_test.sh | head
   Output:
   labex       3669  0.0  0.0  10452  2476 pts/1    S+   04:44   0:00 grep --color=auto --exclude-dir=.bzr --exclude-dir=CVS --exclude-dir=.git --exclude-dir=.hg --exclude-dir=.svn --exclude-dir=.idea --exclude-dir=.tox io_test.sh
   Explanation: Performs a grep check on process status, returning only the active grep process (PID 3669) as the background script process has already terminated.

7. Command: lsof -p 3669
   Output:
   [No output]
   Explanation: Attempts to list open files for PID 3669. Since 3669 was the grep process which exited immediately, no output is returned.

8. Command: ls -l /proc/3669/fd
   Output:
   ls: cannot access '/proc/3669/fd': No such file or directory
   Explanation: Tries to inspect file descriptors of PID 3669 in the /proc directory. It fails because the short-lived grep process has already terminated.

   -q: bytes in POSIX msg queues       819200
   -e: max nice                        0

10. Command: ./io_test.sh >> combined.log 2>&1
    Output:
    [No output or command completed successfully]
    Explanation: Runs the script in the foreground, redirecting both standard output and standard error (via 2>&1) into combined.log in append mode.

12. Command: cat out.log && cat err.log && cat combined.log
    Output:
    STDOUT message
    STDERR message
    STDOUT message
    STDERR message
    Explanation: Prints the contents of out.log (containing only stdout messages), err.log (containing only stderr messages), and combined.log (containing both stdout and stderr messages combined).

13. Command: kill 3669
    Output:
    bash: kill: (3669) - No such process
    Explanation: Attempts to kill process 3669, which fails because the process has already completed and exited.
