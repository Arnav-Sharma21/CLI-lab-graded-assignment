---------------------------------------------------
This section details running a background process and inspecting its open files using lsof.

Command Run:

[1] 3628

3. REDIRECTION DEMONSTRATION
----------------------------
This section demonstrates separating output and error streams versus combining them.

Command Run:
cat out.log && cat err.log

Output:
STDOUT message
STDERR message

Explanation:
Viewing 'out.log' and 'err.log' shows that standard output (FD 1) and standard error (FD 2) were correctly segregated into their respective log files.

Command Run:
./io_test.sh >> combined.log 2>&1 && cat combined.log

Output:
STDOUT message
STDERR message

Explanation:
The operator '2>&1' redirects standard error (FD 2) to the current target of standard output (FD 1), causing both streams to write sequentially into 'combined.log'.

5. TECHNICAL I/O STREAM AND FILE DESCRIPTOR SUMMARY
----------------------------------------------------
In Linux, everything is modeled as a file, and processes interact with files or streams via integer identifiers known as File Descriptors (FDs). 
By default, every process starts with three standardized streams automatically opened:
- File Descriptor 0 (stdin): Standard input stream, typically read from the terminal keyboard or redirected from a file.
- File Descriptor 1 (stdout): Standard output stream, used for normal process messages, typically printed to the terminal screen or redirected to a log.
- File Descriptor 2 (stderr): Standard error stream, reserved for diagnostics, warnings, and error messages to ensure they are visible even when stdout is redirected.
The operating system maintains a per-process file descriptor table mapping these integer values to entries in the global file table. 
Redirections (like '>' and '2>') manipulate these pointers, allowing transparent file writing and inter-process pipelining without altering the program code.
